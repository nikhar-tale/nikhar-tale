## 🏛️ Clean Architecture Approach

I follow a **feature-first, layered architecture** across all Flutter projects:

```
lib/
├── core/                  # Utilities, constants, failure handling
├── features/
│   └── feature_name/
│       ├── data/          # APIs, models, datasources
│       ├── domain/        # Entities, use cases, repositories
│       └── presentation/
│           ├── bloc/      # State management
│           └── ui/        # Widgets, screens
```

> This pattern improves **testability**, **maintainability**, and **team scalability** — enabling parallel feature development without merge conflicts.
